# Realism Patch Generator - 使用说明 (Usage Instructions)

## 项目结构
本项目为 SPT-EFT 的 Realism MOD 补丁生成器。旨在快速生成武器、配件、弹药、装备和消耗品的兼容性 JSON。

### 1. 输入 (Input)
将你想要通过 Realism 模组运行的 **原始 MOD 的物品定义文件 (.json)** 放入 `input` 文件夹。

> 示例：`[2]新酒水-WTT - CornerStore_Food_Items.json`, `[3]新配件-BlackCore.json` 等。

### 2. 输出 (Output)
程序运行后，你的补丁会按照不同的物品类型自动生成在 `output` 文件夹中：
- `weapons.json`: 武器类补丁内容
- `attachments.json`: 配件/导轨/枪口雷指等内容
- `ammo.json`: 弹药/子弹内容
- `gear.json`: 护甲/防弹袋/背包等内容
- `consumables.json`: 医疗品/食物/水分内容

### 3. 设置和模板 (Templates and Configuration)
在项目根目录或程序运行目录中，可以建立 `现实主义物品模板` 文件夹 (Realism Item Templates)。程序会分别加载以下子目录中的 JSON 模板：
- `weapons/`
- `attatchments/`
- `ammo/`
- `gear/`
- `consumables/`

每个模板应当是一个包含 Realism 特定属性（如 `WeapAccuracy`, `BaseTorque` 等）的 JSON 片段。

### 4. 程序运行方式
#### A. 直接运行
直接双击运行 `RealismPatchGenerator.exe`。程序会默认读取 `input` 文件夹并输出到 `output` 文件夹。

#### B. 命令行运行
通过命令行界面 (CMD/PowerShell) 指定自定义输入和输出路径：
```powershell
.\RealismPatchGenerator.exe "C:\Path\To\MyModInput" "C:\Path\To\MyModOutput"
```

## 开发者说明 (For Developers)
- **PatchGenerator.cs**: 包含物品数据解析和格式转换的核心算法。它支持 `VIR`, `STANDARD`, `Clone` 等多种 MOD 设计流派。
- **ItemTypeMappings.cs**: 包含 EFT 物品分类的硬编码对应关系。
- **DefaultTemplates.cs**: 当找不到外部模板文件时，程序使用的内嵌属性默认值。

## 故障排除
1.  **无法识别物品类型**：程序依据 `parentId` 识别类型。如果模组使用了非常规的父 ID，建议在 `ItemTypeMappings.cs` 中补充相应的映射。
2.  **生成文件为空**：请检查 `input` 中的 JSON 格式。如果不是标准的数组或包含物品字典的结构，解析器可能跳过。
